## Исправленные баги (2026-03-30)

- **Персональный код из админки без привязки к пользователю** (`assigned_user_id` = NULL): раньше валидация считала код «чужим». Теперь привязка проверяется только если `assigned_user_id` задан (как у авто-кодов из рассылки).
- **Ввод промо после «Перейти к оплате»** (статус `payment_pending`): кнопка «Ввести промо-код» давала «Заказ не найден». Разрешено для `draft` и `payment_pending`; при смене суммы сбрасывается незавершённый платёж (`db_draft.reset_draft_after_price_change`), пользователь создаёт оплату заново.
