# Пошаговая инструкция: Яндекс 360 для glava.family

## Что получится на выходе

- Почтовые ящики `@glava.family` (hi@, info@, postmaster@ и любые другие)
- Работает через веб-интерфейс mail.yandex.ru и любой почтовый клиент (IMAP/SMTP)
- Бесплатно до 1000 ящиков

---

## Шаг 1. Зарегистрироваться в Яндекс 360

1. Открыть https://360.yandex.ru/
2. Нажать **«Попробовать бесплатно»** (или «Войти» если аккаунт есть)
3. Авторизоваться через Яндекс-аккаунт (личный — подойдёт)
4. На вопрос «Название организации» — ввести `Glava Family` или любое

---

## Шаг 2. Добавить домен

1. В меню слева → **«Домены»** → **«Добавить домен»**
2. Ввести: `glava.family`
3. Яндекс покажет TXT-запись для подтверждения, например:
   ```
   yandex-verification: abc123def456
   ```
   Скопируй значение.

---

## Шаг 3. Найти панель DNS домена

Нужно понять, где настраивается DNS домена glava.family.

**Как узнать:**
```powershell
nslookup -type=NS glava.family
```

Результат покажет NS-серверы. По ним определяем панель:

| NS-серверы | Панель |
|-----------|--------|
| `ns1.timeweb.ru`, `ns2.timeweb.ru` | Timeweb → Домены |
| `ns1.reg.ru`, `ns2.reg.ru` | reg.ru → DNS-записи |
| `ns1.nic.ru`, `ns2.nic.ru` | nic.ru → DNS |
| `ns*.hetzner.com` | Hetzner DNS Console |
| Другие | Сайт регистратора домена |

---

## Шаг 4. Добавить TXT-запись подтверждения

В панели DNS добавить запись:

| Поле | Значение |
|------|---------|
| Тип | TXT |
| Хост / Имя | `@` (или `glava.family`) |
| Значение | `yandex-verification: <КОД_ИЗ_ЯНДЕКС_360>` |
| TTL | 3600 |

Вернуться в Яндекс 360 → нажать **«Проверить»**.
Ждать до 30 минут (обычно 5–10 мин).

---

## Шаг 5. Добавить MX-запись

> ⚠️ Сначала удалить все существующие MX-записи для `@`

| Поле | Значение |
|------|---------|
| Тип | MX |
| Хост | `@` |
| Значение | `mx.yandex.net.` (с точкой в конце) |
| Приоритет | 10 |
| TTL | 3600 |

---

## Шаг 6. Добавить SPF

| Поле | Значение |
|------|---------|
| Тип | TXT |
| Хост | `@` |
| Значение | `v=spf1 include:_spf.yandex.net ~all` |
| TTL | 3600 |

> Если уже есть TXT-запись `v=spf1 ...` — объединить в одну, не создавать вторую.

---

## Шаг 7. Добавить DKIM

1. В Яндекс 360 → **«Домены»** → выбрать `glava.family` → **«DNS-записи»**
2. Скопировать DKIM-ключ (длинная строка `v=DKIM1; k=rsa; p=...`)
3. Добавить в DNS:

| Поле | Значение |
|------|---------|
| Тип | TXT |
| Хост | `mail._domainkey` |
| Значение | (вставить полностью из Яндекс 360) |
| TTL | 3600 |

---

## Шаг 8. Добавить DMARC

| Поле | Значение |
|------|---------|
| Тип | TXT |
| Хост | `_dmarc` |
| Значение | `v=DMARC1; p=none; rua=mailto:postmaster@glava.family` |
| TTL | 3600 |

---

## Шаг 9. Создать ящики

В Яндекс 360 → **«Пользователи»** → **«Добавить пользователя»**:

| Логин | Email | Назначение |
|-------|-------|-----------|
| hi | hi@glava.family | Основной контактный адрес |
| info | info@glava.family | Формы, рассылки |
| postmaster | postmaster@glava.family | DMARC-отчёты, системный |

Для каждого: задать имя, фамилию, пароль.

---

## Шаг 10. Проверка

```powershell
# Запустить скрипт проверки (из корня проекта):
.\tasks\email-glava-family\jobs\verify-dns.ps1
```

Или вручную:
```powershell
nslookup -type=MX glava.family
# → mail exchanger = mx.yandex.net

nslookup -type=TXT glava.family
# → v=spf1 include:_spf.yandex.net ~all
```

Онлайн: https://mxtoolbox.com/SuperTool.aspx → `glava.family` → Email Health

**Финальная проверка:** отправить письмо с `hi@glava.family` на внешний адрес
(Gmail, Mail.ru) и убедиться что:
- письмо дошло
- в заголовках `Authentication-Results` видны `dkim=pass`, `spf=pass`

## Настройка почтового клиента (IMAP)

| Параметр | Значение |
|---------|---------|
| IMAP-сервер | `imap.yandex.ru` |
| IMAP-порт | 993 (SSL) |
| SMTP-сервер | `smtp.yandex.ru` |
| SMTP-порт | 465 (SSL) или 587 (TLS) |
| Логин | полный адрес (`hi@glava.family`) |
| Пароль | пароль пользователя в Яндекс 360 |

> Для входа через сторонний клиент может потребоваться включить «Пароли приложений»
> в Яндекс ID: https://id.yandex.ru → Безопасность → Пароли приложений
