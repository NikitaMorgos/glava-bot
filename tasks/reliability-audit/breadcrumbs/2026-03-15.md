# 2026-03-15 — Аудит надёжности: первичный проход

## Что сделано

1. Полный SRE/reliability/recoverability аудит проекта GLAVA.
2. Проверены 50+ файлов по 4 направлениям:
   - Error handling, resilience, retries, graceful shutdown
   - DB layer, storage, sessions, connection management
   - Deploy, monitoring, systemd, Docker, Nginx
   - External service dependencies и failure impact
3. Найдено 27 рисков: 5 Critical, 8 High, 10 Medium, 4 Low.
4. Идентифицировано 10 single points of failure.
5. Проанализировано 10 сценариев отказа с последствиями.
6. Проверено 15 readiness-механизмов (0 полностью реализованы).
7. Оценено 11 внешних сервисов по timeout/retry/fallback.
8. PDF-отчёт в `docs/RELIABILITY_AUDIT_2026-03-15.pdf`.

## Ключевые находки

- **Всё на одном VPS** — полный SPOF от железа до приложения.
- **check_payment** глотает ошибки API и возвращает "pending" — пользователь может застрять навсегда.
- **daemon=True потоки** убиваются при restart без cleanup.
- **n8n pipeline** — самая хрупкая часть: 7 LLM-вызовов без error branch, без retry.
- **Pipeline failures invisible** — пользователь не узнаёт о провале генерации биографии.
- **Нет ни одного из 15 стандартных readiness-механизмов** полностью.
- **DB без pooling** — каждый запрос = TCP connect; burst → connection exhaustion.
