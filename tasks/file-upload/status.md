### Исправленные баги
- **v1.0 → v1.1 (2026-03-29):** Токены хранились in-memory — бот и кабинет работают в разных процессах, кабинет не видел токены бота. Перенесено в PostgreSQL (таблица `upload_tokens`). Добавлен `_maybe_ensure_table()` при валидации токена (кабинет).
- **v1.1 → v1.2 (2026-03-30):** Транскрипция после веб-загрузки не запускалась: `ImportError` — код вызывал несуществующую функцию `run` вместо `run_pipeline_sync`. Файл загружался в S3 и записывался в БД, но транскрипция падала молча в daemon-потоке. Исправлено имя функции, добавлено логирование в фоновый поток. Gunicorn timeout увеличен 120→600 сек (загрузка больших файлов на медленном мобильном интернете).

### Как работает для пользователя

1. Пользователь отправляет аудио/документ > 20 МБ → бот автоматически отвечает кнопкой «Загрузить через браузер»
2. Пользователь отправляет видео → то же самое
3. Пользователь вводит `/upload` → получает ссылку вручную
4. По ссылке — страница с drag & drop, прогресс-баром, валидацией формата
5. После загрузки — файл в S3, запись в БД, уведомление в Telegram, запуск транскрипции
