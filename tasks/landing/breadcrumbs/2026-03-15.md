## Что было сделано

### Итерация 1 — MVP по ТЗ

- Прочитан `AGENTS.md`: роль Агента Б, правила лендинга в `landing/`, домен `glava.family`
- Прочитан `.docx` через Python (zipfile + xml) — извлечён полный текст ТЗ
- Написан полный лендинг по ТЗ с нуля: все блоки, карусель, FAQ, мобиль
- Заметки по решениям:
  - Карусель: данные вынесены в JS-массив `REVIEWS`, чтобы добавление отзывов не требовало правки HTML
  - FAQ: данные в `FAQ`, accordion через toggle класса `.open`
  - Книга в hero: CSS-слои без картинок, SVG-иконки

### Итерация 2 — Дизайн-апгрейд

**Проблемы исходной версии, которые решали:**
- Container слишком узкий (1100px) — страница выглядела «боязливо» на десктопе
- Мелкий body-текст в контентных блоках
- Hero недостаточно выразителен
- Слишком мягкий контраст, бледные карточки

**Ключевые правки:**
- Container 1280px + breakpoint 1440px
- html font-size 17px
- Все типографические размеры пересмотрены через clamp()
- CSS-переменные палитры обогащены
- Trust-маркеры добавлены в hero
- Декоративный ornament в Final CTA

### Итерация 3 — UX-апгрейд (конверсия)

**Проблемы после итерации 2:**
- На десктопе бургер-меню выглядел как для мобиля — нет видимой структуры
- Hero текст абстрактный, один путь входа (только один CTA)
- FAQ в неверном порядке (цена первой, а не результат)
- После отзывов нет конверсионной точки
- Блок ценностей — описания в 1 предложение, не отвечают «почему это важно»

**Ключевые правки:**
- Desktop nav: `.nav-links` + `.nav-cta` (видны ≥1025px), бургер только мобиль
- Hero: новый слоган, desc конкретизирован, цена отдельной строкой, secondary CTA
- Values: заголовок секции, развёрнутые описания
- Steps: новый заголовок «Всё проще, чем кажется», reassurance note
- Mid-CTA: новый блок после отзывов (тёмный, emotional hook)
- FAQ: реорганизован в конверсионный порядок
- Final CTA: what-you-get чеклист
- Book-заглушка: убрана из hero по запросу заказчика

## Решения, которые стоит помнить

**Почему нет book-визуала в hero?**  
Заказчик попросил убрать — реальная обложка ещё в работе. Место зарезервировано, добавим когда придёт фото. Нужно вернуть `.hero-visual` блок + двухколоночный grid в `.hero-inner`.

**Как добавить отзыв:**
```javascript
// В landing/index.html, в массиве REVIEWS:
{
    text: '«Текст отзыва...»',
    name: 'Имя',
    role: 'Роль / кем приходится',
},
```

**Как добавить вопрос в FAQ:**
```javascript
// В landing/index.html, в массиве FAQ:
{
    q: 'Вопрос?',
    a: `<p>Ответ.</p>`,
},
```

**Как вернуть книгу в hero:**  
Нужно восстановить двухколоночный grid в `.hero-inner` и добавить `.hero-visual` блок.  
Шаблон был (`grid-template-columns: 55fr 45fr; gap: 5rem`).
