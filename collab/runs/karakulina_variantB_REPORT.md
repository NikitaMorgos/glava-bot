## Наблюдения и рекомендации

### 🟢 Важное наблюдение: Proofreader работает при объёме ≤10K симв

В этом прогоне Корректор **впервые отработал корректно** — 31 правка, 4/5 глав, `Готово к вёрстке: True`. Объём на входе был ~9,957 симв.

В прогонах с двумя TR (~22K симв на входе Stage3) Корректор системно возвращал пустой JSON.

**Гипотеза:** Корректор падает при входе >~15K симв — превышает эффективный контекст для генерации полного текста в ответ. Нужно проверить на промежуточных объёмах.

**Рекомендация:** Добавить разбивку на главы в Корректоре — обрабатывать по одной главе итеративно. Это снимет ограничение объёма и даст стабильный результат.

---

### 🟡 Strict Gates блокируют Вариант B

Strict Gates настроены на полный объём книги (~20K+ симв) и блокируют Stage2 и Stage3 при TR1-only основе (~10K симв). Для тестовых прогонов Варианта B это ложное срабатывание.

**Рекомендация:** Добавить режим `--variant-b` или `--phase-b-expected` в Strict Gates — при этом флаге проверять не абсолютный объём, а относительный рост (Phase B должна дать +20% минимум).

Оставшиеся 4 ошибки — типичный «потолок» для Phase B без нормализованного fact_map по TR2. FC находит несоответствия, которые Писатель не может исправить без знания всех фактов из TR2.
