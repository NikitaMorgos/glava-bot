# GLAVA Collab — протокол совместной работы

Эта папка — общая рабочая среда для **Даши** (продакт-менеджер, работает в Claude)
и **разработчика** (работает в Cursor). Здесь мы ставим задачи, согласуем спеки
и фиксируем результаты без таск-трекера.

---

## Как это работает

```
Даша пишет спек → меняет статус на "cursor-working"
        ↓
Разработчик видит файл → Cursor читает спек → реализует
        ↓
Cursor пишет результат → меняет статус на "dasha-review"
        ↓
Даша проверяет → пишет комментарий → меняет статус на "done" или "cursor-working"
```

Уведомления — через Dropbox (файл изменился) или Telegram (договорённость отдельно).

---

## Структура папки

```
collab/
  README.md                   ← этот файл (протокол)
  context/
    glava-for-dasha.md        ← контекст GLAVA для Даши и её Claude
  tasks/
    _template.md              ← шаблон новой задачи
    001-название-задачи.md    ← задачи нумеруются
    002-...
```

---

## Статусы задачи

| Статус | Кто меняет | Значение |
|--------|-----------|----------|
| `new` | Даша | Спек написан, ждёт разработчика |
| `cursor-working` | Даша / Cursor | Разработчик взял в работу |
| `dasha-review` | Cursor | Реализовано, ждёт проверки Даши |
| `iteration-N` | Даша | Нужна правка, N = номер итерации |
| `done` | Оба | Задача закрыта |
| `on-hold` | Оба | Пауза (причина в задаче) |

---

## Правила для Даши

1. **Создавай задачу из шаблона** (`tasks/_template.md`) — скопируй, переименуй, заполни
2. **Номер задачи** — следующий по порядку (001, 002, ...)
3. **Спек** — пиши конкретно: что должно получиться, как проверить, что нельзя ломать
4. **Итерации** — пиши комментарий в раздел «Комментарии», меняй статус на `iteration-N`
5. **Контекст для Claude** — загрузи `context/glava-for-dasha.md` в свой Claude Project один раз

---

## Правила для Cursor

1. **При открытии задачи** — прочитай спек полностью, особенно «Ограничения»
2. **Реализация** — описывай кратко: что сделано, какие файлы изменены, как проверить
3. **Не меняй спек** — только раздел «Реализация» и «Комментарии»
4. **Статус** — меняй на `dasha-review` после завершения
5. **Если спек неясен** — пиши вопрос в «Комментарии», статус оставляй `cursor-working`

---

## Типы задач, которые хорошо сюда подходят

- Правка или написание промптов (Stage 1–4, CCO)
- Изменение сценария бота (тексты сообщений, логика веток)
- Добавление нового агента в пайплайн (Даша — промпт, Cursor — код)
- Изменение CCO-скрипта диалога
- Новые шаблоны писем / уведомлений
- Согласование поведения агентов (что принять, что отклонить)

---

## Что сюда НЕ подходит

- Хотфиксы и баги (слишком быстро, не нужен протокол)
- Инфраструктурные задачи без PM-части
- Срочные правки (лучше Telegram → потом задокументировать сюда)
