# Текущее состояние пайплайна GLAVA

> Живой документ. Обновляется при каждом значимом изменении.
> Последнее обновление: 2026-04-07
>
> Источник правды по версиям: `prompts/pipeline_config.json`
> Источник правды по промптам: `prompts/*.md`

---

## Общая схема

```
Голосовые / фото
      ↓
[01] Очиститель          — чистит ASR-транскрипт
      ↓ cleaned_transcript.txt
[02] Фактолог            — извлекает факты → fact_map.json
      ↓ fact_map              ↓ fact_map (параллельно)
[03] Писатель   ←——[12] Историк-краевед  — вплетает контекст эпохи
      ↓ book_draft.json
[04] Фактчекер  ——→ (цикл возврата до 3 раз) ——→ [03] Писатель
      ↓ (verdict: pass)
[05] Литредактор ——→ (цикл возврата до 2 раз) ——→ [03] Писатель
      ↓ edited_draft
[06] Корректор           — финальная вычитка
      ↓ final_text

Параллельно с вёрсткой:
[11] Интервьюер          — генерирует уточняющие вопросы + блиц

[07] Фоторедактор        — анализирует фото, создаёт подписи
      ↓ processed_photos
[15] Арт-директор        — планирует раскладку страниц
      ↓ page_plan (art_director_result)
[08] Верстальщик         — верстает макет книги в код
      ↓ layout_result
[09] QA вёрстки  ——→ (цикл возврата до 3 раз) ——→ [08] Верстальщик
      ↓ (verdict: pass)
[13] Дизайнер обложки    — создаёт задание на обложку
      ↓ cover_spec
      PDF
```

## Агенты: текущие версии и роли

### [01] Очиститель · `01_cleaner_v1.md`
**Модель:** claude-haiku (быстрый, дешёвый)  
**Роль:** Чистит сырой ASR-транскрипт — пунктуация, исправление имён, разметка спикеров.  
**Вход:** raw_text, subject_name, narrator_name  
**Выход:** cleaned_text (строка)  
**Ограничения:** НЕ меняет содержание, НЕ добавляет слова. Только форматирование.  
**Что можно менять в промпте:** правила исправления ASR-ошибок, формат разметки спикеров

---

### [02] Фактолог · `02_fact_extractor_v3.3.md`
**Модель:** claude-sonnet
**Роль:** Извлекает все факты из интервью: персоны, события, хронология, цитаты, места, характер, gaps.
**Вход:** cleaned_transcript, subject info, existing_facts (null в Фазе A, данные в Фазе B)
**Выход:** `fact_map.json`
```json
{
  "subject": {"name", "birth_year", "death_year", "birth_place"},
  "persons": [{"id", "name", "relation_to_subject", "access_status"}],
  "timeline": [{"id", "date", "title", "life_period", "confidence"}],
  "locations": [...],
  "quotes": [{"text", "type": "direct|indirect", "usable_in_book"}],
  "character_traits": [...],
  "gaps": [{"period", "description", "suggested_questions"}],
  "conflicts": [...]
}
```
**Ключевые правила:** НЕ интерпретирует, НЕ додумывает. Только то, что явно сказано. **death_year** заполняется ТОЛЬКО если смерть явно названа в интервью — нельзя выводить из последнего события.
**Что можно менять:** критерии извлечения, формат цитат, правила определения gaps

---

### [12] Историк-краевед · `12_historian_v3.md`
**Модель:** claude-sonnet (temp 0.3)  
**Роль:** Параллельно с Писателем обогащает хронологию контекстом эпохи — что происходило в стране/регионе пока жил герой.  
**Вход:** fact_map (timeline + locations)  
**Выход:** `historian_result.json` — suggested_insertions (исторические вставки для Писателя)  
**Ограничения:** confidence обязателен. Строгий критерий мировых событий — только прямая связь с жизнью героя.  
**Что можно менять:** критерии отбора событий, глубина исторического контекста

---

### [03] Писатель · `03_ghostwriter_v2.6.md`
**Модель:** claude-sonnet (temp 0.6 — самый творческий)
**Роль:** Пишет биографический текст книги из фактов.
**Вход:** fact_map + historian_result + transcripts
**Выход:** `book_draft.json`
```json
{
  "chapters": [
    {
      "id": "ch_01 | ch_02 | ch_03 | ch_04 | epilogue",
      "title": "...",
      "covers_period": "...",
      "timeline_period": "...",
      "paragraphs": [{"text", "source_refs", "timeline_period"}],
      "callouts": [{"type": "pull-quote|hist_block", "text"}]
    }
  ]
}
```
**Структура книги (v2.6):**
- ch_01 — Основные даты жизни (bio_data + timeline, awards в bio_data, НЕ в timeline)
- ch_02 — История жизни (нарратив + исторические справки, БЕЗ цитат-выносок)
- ch_03 — Портрет человека (характер + ценности, объединены из бывших гл.03 и гл.05)
- ch_04 — Интересные факты и жизненные истории
- epilogue — Обобщение жизни (3–5 абзацев, объективно, без оценок)

**Правила размещения:**
- Исторические справки — ТОЛЬКО в ch_02, ≥50% страниц главы, макс 1 на страницу
- Цитаты (callout) — ТОЛЬКО в ch_03 и ch_04, НЕ в ch_02
- death_year на обложке — только из subject.death_year (Фактолог), если null → «род. ГГГГ»

**Что можно менять:** стиль изложения, правила callout, структуру глав

---

### [04] Фактчекер · `04_fact_checker_v2.md`
**Модель:** claude-sonnet (temp 0.1 — точный)  
**Роль:** Проверяет что текст книги не противоречит исходным фактам.  
**Вход:** book_draft + fact_map + transcripts  
**Выход:** `{"verdict": "pass|fail", "issues": [...], "warnings": [...]}`  
**Цикл:** При fail → Писатель переписывает → снова Фактчекер. До 3 итераций.  
**Ключевые правила:** fact_map приоритетнее transcript для топонимов. confidence_upgrade тип ошибки.  
**Что можно менять:** строгость проверки, типы ошибок, iteration_strictness

---

### [05] Литредактор · `05_literary_editor_v3.md`
**Модель:** claude-sonnet (temp 0.5)  
**Роль:** Шлифует стиль и форму текста. НЕ меняет факты.  
**Вход:** book_draft (после Фактчекера) + fact_checker_warnings  
**Выход:** отредактированный book_draft (та же структура)  
**Цикл:** Может вернуть Писателю если структурная проблема. До 2 итераций.  
**Ключевые правила:** эпистемические маркеры, callout обязателен для ch_03/ch_04, 2-3 исторические вставки.
**Что можно менять:** правила стиля, требования к callout, баланс исторических вставок

---

### [06] Корректор · `06_proofreader_v1.md`
**Модель:** claude-sonnet (temp 0.0 — максимально точный)  
**Роль:** Финальная вычитка перед вёрсткой. Орфография, пунктуация, типографика.  
**Вход:** отредактированный book_draft  
**Выход:** финальный book_draft + proofreader_report (passport стиля)  
**Ограничения:** После него текст уходит в PDF. Ноль терпимости к ошибкам.  
**Что можно менять:** правила типографики, паспорт стиля, список исключений

---

### [07] Фоторедактор · `07_photo_editor_v1.md`
**Модель:** claude-sonnet (vision=true)  
**Роль:** Анализирует фото, создаёт подписи, привязывает к периодам жизни.  
**Вход:** фотографии + fact_map  
**Выход:** `processed_photos` — список с подписями, датами, привязкой к главам  
**Ограничения:** Без фото — возвращает пустой processed_photos, не падает.  
**Что можно менять:** формат подписей, критерии привязки к периодам

---

### [15] Арт-директор · `15_layout_art_director_v1.8.md`
**Модель:** claude-sonnet (vision=true, temp 0.45)
**Роль:** Планирует раскладку страниц: где текст, где фото, какой приём размещения.
**Вход:** final_text (главы) + processed_photos + book_structure
**Выход:** `page_plan` — постраничный план макета
**Ключевые правила (v1.8):**
- Вертикальное фото → 1 на страницу, максимальная высота
- Горизонтальные фото → по 2 на страницу (pair_stack), группировать вместе
- Микс запрещён: вертикальное + горизонтальное на одной странице — НИКОГДА
- pair_side ЗАПРЕЩЁН для вертикальных
- Хронологический порядок, маппинг по timeline_period

**Что можно менять:** правила размещения по типу фото, обязательные элементы глав

---

### [08] Верстальщик · `08_layout_designer_v3.14.md`
**Модель:** claude-sonnet (max_tokens=64000 — самый большой)  
**Роль:** Реализует page_plan в layout-код (JSON-структура страниц для PDF-рендерера).  
**Вход:** page_plan + final_text + processed_photos  
**Выход:** `layout_result` с layout_code  
**Ключевые правила:** callout/hist_block через Table. `##` как 18pt section_header. hist_block фон #f0ebe0. titulny_razvorot обязателен.  
**Что можно менять:** визуальные правила, типографические стили

---

### [09] QA вёрстки · `09_qa_layout_v1.md`
**Модель:** claude-sonnet (vision=true, temp 0.05)  
**Роль:** Проверяет PDF-макет визуально.  
**Вход:** PDF-страницы (изображения) + ожидаемый layout  
**Выход:** `{"verdict": "pass|fail", "issues": [{"type", "owner": "layout_designer|builder"}]}`  
**Цикл:** При fail → Верстальщик правит → снова QA. До 3 итераций. Builder-only issues → принимаем как tech debt.

---

### [13] Дизайнер обложки · `13_cover_designer_v2.6.md`
**Модель:** claude-sonnet (vision=true, temp 0.65)
**Роль:** Выбирает главное фото, составляет задание на тушевой портрет, оценивает результат, создаёт финальную композицию.
**Вход:** processed_photos + subject info + book mood
**Выход:** `cover_spec` — 3-зонная композиция A/B/C, 3 текстовые строки
**Ключевые правила:** строгая 3-зона. УБРАНА декорация логотипа/года. 2-й валидационный вызов обязателен. death_year — ТОЛЬКО из subject.death_year (Фактолог), если null → «род. ГГГГ», запрещено брать год из текста биографии.

---

### [11] Интервьюер · `11_interview_architect_v4.1.md`
**Модель:** claude-sonnet (temp 0.5)  
**Роль:** Генерирует уточняющие вопросы для клиента на основе gaps и структуры книги.  
**Вход:** fact_map (gaps, timeline, persons) + book_chapters_summary + blitz_questions[]  
**Выход:** `interview_questions` (12–18 вопросов) + собирается в `ai_questions = {clarifying, blitz}`  
**Ключевые правила:** дедуп с блиц (шаг 7.5). Проверка критических фактов — death_year, birth_place, разрывы >5 лет (шаг 8.5). Вызывается ОДНОКРАТНО.  
**Что можно менять:** лимит вопросов, категории, правила дедупликации

## Циклы возврата

| Цикл | Условие возврата | Макс. итераций | Что происходит при превышении |
|------|-----------------|----------------|-------------------------------|
| Фактчекер → Писатель | verdict: fail | 3 | Эскалация (не реализована, принимаем как есть) |
| Литредактор → Писатель | структурная проблема | 2 | Принимаем текущий вариант |
| QA → Верстальщик | verdict: fail, layout issues | 3 | Эскалация к Продюсеру |

## Блиц-вопросы

34 фиксированных вопроса о характере и ценностях героя.  
Хранятся в `pipeline_config.json` → `blitz_questions[]`.  
Доставляются клиенту вместе с уточняющими вопросами Интервьюера в `ai_questions.blitz`.  
**Изменить список:** отредактировать `pipeline_config.json`, задеплоить через API.

## Что можно менять без разработчика

| Изменение | Как |
|-----------|-----|
| Текст и правила промпта агента | Редактировать `.md` → `POST /api/prompts/<role>/upload` |
| Список блиц-вопросов | Редактировать `pipeline_config.json` → деплой |
| Модель агента | `pipeline_config.json` → деплой |
| Температуру агента | `pipeline_config.json` → деплой |
| max_tokens агента | `pipeline_config.json` → деплой |

## Что требует разработчика

| Изменение | Почему |
|-----------|--------|
| Новый агент в цепочке | Нужно добавить вызов в `pipeline_utils.py` |
| Изменить формат JSON между агентами | Ломает следующий агент в цепочке |
| Изменить логику цикла (макс. итераций) | Константы в коде |
| Добавить новое поле в `pipeline_config.json` | Нужно читать это поле в коде |
| Изменить порядок агентов | Оркестрация в коде |

## Как предложить изменение процесса

Создать задачу в `collab/tasks/`:

```markdown
## Текущее поведение
[Что делает агент/процесс сейчас — конкретно]

## Проблема / наблюдение
[Что не работает или что хочется улучшить — с примером]

## Предложение
[Что изменить и как это должно работать]

## Затрагивает
- [ ] Только промпт (Даша делает сама)
- [ ] pipeline_config.json (Даша делает сама)
- [ ] Код — нужен разработчик
```

## Как обновлять этот документ

При любом значимом изменении пайплайна — обновить соответствующий раздел:
- Новая версия промпта → обновить строку версии и ключевые правила агента
- Новый агент → добавить раздел
- Изменение формата данных → обновить схему Вход/Выход
- Изменение цикла → обновить таблицу циклов
