## Для Даши — что делает этот агент

### Роль
Берёт очищенный транскрипт интервью и извлекает из него структурированную карту фактов: хронологию жизни, людей, места, характер, пробелы (gaps) и противоречия.  
Это **основа всего пайплайна** — качество всех последующих этапов зависит от полноты fact_map.

### Что должно быть в хорошем результате
- [ ] Все факты из транскрипта извлечены, ни один значимый эпизод не потерян
- [ ] Хронология без внутренних противоречий (3-шаговая проверка)
- [ ] Каждый человек в `persons[]` имеет `access_status` (живой/умер/недоступен)
- [ ] `gaps[]` заполнены — агент явно называет что неизвестно
- [ ] Прямая речь из ASR помечена маркерами и не приписана неверному спикеру
- [ ] Детали локаций конкретные (не «деревня», а «село Мариевка, Кировоградская обл.»)

### Типичные ошибки и как их ловить
| Симптом | Вероятная причина | Что поправить в промпте |
|---------|-------------------|------------------------|
| В fact_map нет ярких эпизодов (быки, крапива и т.п.) | Агент извлекает только «сухие» факты, игнорирует нарративные сцены | Добавить правило: «сохраняй живые эпизоды и анекдоты — они важнее абстрактных описаний» |
| `gaps[]` пустой | Промпт не требует явно формулировать пробелы | Усилить правило gaps: «если что-то не упомянуто в транскрипте — это gap» |
| Прямая речь приписана не тому спикеру | ASR не разделяет голоса чётко | Добавить инструкцию по маркерам ASR-диаризации |
| Хронология содержит невозможные даты | Агент «вычисляет» даты по контексту неверно | Добавить правило: «при неуверенности — пиши диапазон, не домысливай» |

### На что смотреть при написании промпта
- **Обязательно:** список чеклистов (известные факты о герое для верификации)
- **Нельзя убирать:** 3-шаговую проверку хронологии — без неё агент создаёт противоречия
- **Помогают примеры:** показать в промпте как выглядит хорошо заполненный `gaps[]` и плохо

### Как проверить результат вручную
1. Открыть `exports/project_fact_map_*.json`
2. Проверить: `facts[]` — не менее 80–100 фактов для часового интервью
3. Проверить: `gaps[]` — должны быть, иначе интервьюер не задаст нужных вопросов
4. Проверить: `persons[]` — все упомянутые люди с `access_status`
5. Красный флаг: `conflicts[]` пустой при длинном интервью — скорее всего агент не проверял

## Для разработки — техническая спецификация

### Место в пайплайне
```
[01 Cleaner] → [02 FACT EXTRACTOR] → [Stage 2: Historian]
                                    → [Stage 4: Interview Architect]
```
> ⚠️ `gaps[]` из fact_map нужны для Interview Architect (Stage 4). Передавай в Stage 4 именно выход Fact Extractor, а не historian_extended.json — у историка нет поля `gaps`.

### Входные данные

```json
{
  "project_id": "string",
  "cleaned_transcript": "string — очищенный транскрипт",
  "character_name": "string",
  "narrator_name": "string",
  "narrator_relation": "string — кем приходится герою",
  "known_birth_year": "number | null",
  "known_details": "string — известные факты для верификации",
  "checklist": ["список фактов которые должны быть в fact_map"]
}
```

### Выходные данные

```json
{
  "facts": [{"id": "F001", "text": "...", "source": "transcript|inferred", "confidence": "high|medium|low"}],
  "persons": [{"name": "...", "relation": "...", "access_status": "alive|deceased|unknown|unreachable"}],
  "timeline": [{"period": "1941-1945", "events": ["..."]}],
  "character_traits": [{"trait": "...", "evidence": "..."}],
  "locations": [{"name": "...", "period": "...", "context": "..."}],
  "gaps": [{"id": "G001", "description": "Неизвестно...", "importance": "high|medium|low"}],
  "conflicts": [{"id": "C001", "description": "Противоречие между..."}],
  "checklist_coverage": "0-100%"
}
```

### Конфигурация в `pipeline_config.json`

```json
{
  "fact_extractor": {
    "model": "claude-sonnet-4-20250514",
    "max_tokens": 32000,
    "temperature": 0.15,
    "prompt_file": "02_fact_extractor_v3.2.md"
  }
}
```

**Почему такие параметры:**
- `temperature: 0.15` — почти детерминированный: факты не должны «придумываться»
- `max_tokens: 32000` — при 40+ мин интервью fact_map может быть большим; при 16000 обрезался

### Как вызывается в коде

```python
# scripts/test_stage1_*.py
from pipeline_utils import load_config, load_prompt, call_agent

cfg = load_config()
system_prompt = load_prompt(cfg["fact_extractor"]["prompt_file"])
user_message = json.dumps({
    "project_id": PROJECT_ID,
    "cleaned_transcript": cleaned_text,
    "character_name": CHARACTER_NAME,
    ...
}, ensure_ascii=False)
result, in_tok, out_tok = await call_agent(
    client, "FACT_EXTRACTOR", "fact_extractor",
    system_prompt, user_message, cfg
)
```

### Известные ограничения
- При транскрипте > 40 000 символов может обрезать факты в конце — в этом случае делить транскрипт на части
- Не умеет надёжно определять возраст человека по косвенным признакам — лучше передавать `known_birth_year`
