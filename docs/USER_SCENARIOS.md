# Пользовательские сценарии и тесты бота GLAVA

## 1. Схема пользовательских сценариев

```
                                    ┌─────────────────┐
                                    │   /start        │
                                    │   Приветствие   │
                                    └────────┬────────┘
                                             │
              ┌──────────────────────────────┼──────────────────────────────┐
              ▼                              ▼                              ▼
     ┌────────────────┐            ┌────────────────┐            ┌────────────────┐
     │ Посмотреть     │            │ Узнать         │            │ Начать          │
     │ пример        │            │ стоимость      │            │ (intro_start)   │
     └───────┬────────┘            └───────┬────────┘            └────────┬───────┘
            │ ← Назад                      │ ← Назад                       │
            └─────────────────────────────┴───────────────────────────────┘
                                                          │
                                    ┌─────────────────────┼─────────────────────┐
                                    ▼                     ▼                     ▼
                           [черновик создан]    [payment_pending]        [paid]
                                    │                     │                     │
                                    ▼                     │                     │
                           Персонажи (0)                  │              «Оплата прошла.
                           → Ввод имени (без кнопок)      │               /list»
                           → Ввод родства                 │                     │
                                    │                     ▼                     │
                                    ▼              «Ожидаем оплату.             │
                           Персонажи (список) +           Проверить оплату»     │
                           кнопки [Добавить ещё,          │                     │
                           Продолжить, Удалить]           │                     │
                                    │                     │                     │
                                    ▼                     │                     │
                                    │                     │ (payment_check)     │
                                    ▼                     ▼                     │
                           Email для чека            check_payment ──────────────┘
                                    │                     │ paid → set_draft_paid
                                    ▼                     │
                           Итого заказ                    │
                           • Перейти к оплате (order_pay)  │
                           • Назад / Изменить              │
                                    │                     │
                                    ▼                     │
                           create_payment → payment_url   │
                           «Открыть оплату» / «Проверить оплату»
                                    │
                                    └─────────────────────► (см. payment_pending выше)

После оплаты (status = paid):
                                    ┌─────────────────────────────────────────┐
                                    │ Голосовые / Аудио / Документ (аудио)    │
                                    │ → сохранение → транскрипция (mymeet/    │
                                    │   plaud/speechkit) → LLM bio             │
                                    ├─────────────────────────────────────────┤
                                    │ Фото → сохранение → запрос подписи       │
                                    │ → текст → подпись сохранена             │
                                    ├─────────────────────────────────────────┤
                                    │ /list → список голосовых и фото со      │
                                    │         ссылками на скачивание          │
                                    ├─────────────────────────────────────────┤
                                    │ /cabinet → ввод пароля (≥6 символов)     │
                                    │ → пароль сохранён, вход: cabinet.glava   │
                                    └─────────────────────────────────────────┘

До оплаты:
  • Голос/аудио/фото → «Сначала оформите заказ» + кнопка «Начать»
  • Текст принимается как имя/родство/email только в соответствующих state'ах FSM; иначе → «Используйте кнопки меню или /start»
```

## 2. Таблица сценариев

| № | Сценарий | Шаги пользователя | Ожидаемый результат |
|---|----------|-------------------|----------------------|
| 1 | Старт | `/start` | Приветствие + кнопки: Пример, Стоимость, Начать |
| 2 | Просмотр примера | Пример → Назад | Текст примера книги → возврат в главное меню |
| 3 | Просмотр стоимости | Узнать стоимость → Назад | Цена (990 руб/персонаж) → главное меню |
| 4 | Оформление заказа (новый) | Начать → (бот просит имя) → имя → (бот просит родство) → родство → (появляются кнопки) → Продолжить → email → Итого | Черновик создан. При 0 персонажах — сразу ввод имени без кнопок; после первого персонажа — кнопки «Добавить ещё», «Продолжить»; email, экран «Итого» с кнопкой «Перейти к оплате» |
| 5 | Редактирование персонажей | В конфиге персонажей → Изменить список → удалить → Назад | Список обновлён или возврат к списку |
| 6 | Переход к оплате | На экране «Итого» → Перейти к оплате | create_payment, ссылка на оплату, кнопки «Открыть оплату», «Проверить оплату» |
| 7 | Проверка оплаты (ещё не оплачено) | После перехода к оплате → Проверить оплату | Алерт «Оплата ещё не поступила» (при заглушке — всегда pending) |
| 8 | Проверка оплаты (оплачено) | После реальной оплаты (или ручной смены статуса) → Проверить оплату | «Оплата прошла! Можете отправлять голосовые и фото. /list» |
| 9 | Возврат с экрана оплаты | На экране оплаты → Назад | Возврат к экрану «Итого» заказа |
| 10 | Голосовое до оплаты | Отправить голосовое (без paid) | «Сначала оформите заказ» + кнопка Начать |
| 11 | Голосовое после оплаты | Отправить голосовое (paid) | «Аудио сохранено», запуск пайплайна транскрипции |
| 12 | Аудио/документ после оплаты | Отправить аудиофайл или документ .ogg/.mp3 (paid) | Аналогично голосовому — сохранение + транскрипция |
| 13 | Фото до оплаты | Отправить фото (без paid) | Блокировка, предложение оформить заказ |
| 14 | Фото после оплаты | Отправить фото (paid) → затем текст подписи | «Фото сохранено. Напиши подпись» → «Подпись сохранена.» |
| 15 | Список материалов | `/list` (после оплаты, с материалами) | Список голосовых и фото со ссылками на скачивание |
| 16 | Список пустой | `/list` без материалов | «Пока ничего нет. Отправь голосовые или фото после оплаты.» |
| 17 | Кабинет | `/cabinet` → ввод пароля ≥6 символов | «Пароль сохранён. cabinet.glava.family» |
| 18 | Кабинет — короткий пароль | `/cabinet` → пароль &lt;6 символов | «Пароль от 6 символов. /cabinet» |
| 19 | Произвольный текст | Написать текст вне state'ов имя/родство/email/подпись/пароль | «Используйте кнопки меню или /start» (текст принимается только в соответствующих state'ах FSM) |
| 20 | Неверный email | В режиме ввода email ввести не-email | «Похоже, email введён неверно» |
| 21 | Возобновление черновика | /start при наличии draft → Начать | Продолжение: если characters=[], возврат в режим ввода имени (без кнопок); иначе — список персонажей с кнопками, или email, или итого (в зависимости от состояния черновика) |
| 22 | Возобновление ожидания оплаты | /start при payment_pending → Начать | Экран «Ожидаем подтверждение оплаты» с кнопкой «Проверить оплату» |

## 3. Результаты автотестов

Автотесты (с моками БД и Telegram API) лежат в **`tests/test_bot_flows.py`**.

### Как запустить

Из корня проекта (где лежит `main.py`):

```bash
pip install pytest pytest-asyncio
pytest tests/test_bot_flows.py -v
```

Или с указанием каталога:

```bash
pytest tests/ -v
```

### Покрытие сценариев тестами

| Тест | Сценарий |
|------|----------|
| `test_cmd_start_sends_intro` | Старт: приветствие и меню |
| `test_callback_intro_main` | Кнопка «Назад» → главное меню |
| `test_callback_intro_example` | Просмотр примера |
| `test_callback_intro_start_creates_draft` | Начать → черновик; при 0 персонажах — запрос имени (без кнопок) |
| `test_callback_intro_start_payment_pending` | Начать при payment_pending → экран оплаты |
| `test_callback_intro_start_paid` | Начать при paid → «Оплата прошла» |
| `test_callback_order_pay_calls_create_payment` | Перейти к оплате → create_payment, ссылка |
| `test_callback_payment_check_paid` | Проверить оплату при paid → set_draft_paid, успех |
| `test_voice_without_paid_blocked` | Голосовое до оплаты — блокировка |
| `test_list_empty` | /list при пустом списке |
| `test_cabinet_sets_awaiting_password` | /cabinet — запрос пароля |
| `test_text_unknown_no_prepay_reply` | Произвольный текст → «Используйте кнопки» |
| `test_prepay_character_name_then_relation` | Ввод имени персонажа → запрос родства |
| `test_prepay_email_invalid_reply` | Неверный email → сообщение об ошибке |
| `test_photo_without_paid_blocked` | Фото до оплаты — блокировка |

**Ожидаемый результат запуска:** все перечисленные тесты проходят (например, 14–15 passed). Ошибки окружения (нет pytest, нет доступа к БД без моков) в этом файле не учитываются — тесты рассчитаны на моки.

## 4. Краткая схема по типам ввода

| Ввод | До оплаты | После оплаты |
|------|-----------|--------------|
| `/start` | Приветствие + меню | То же |
| `/list` | Список (может быть пустой) | Голосовые и фото со ссылками |
| `/cabinet` | Запрос пароля для кабинета | То же |
| Кнопки (callback) | Навигация prepay, оплата | Навигация, проверка оплаты |
| Текст | Имя/родство/email (если в режиме prepay), иначе «кнопки»; пароль кабинета | Подпись к фото (если есть ожидающее фото), иначе «кнопки» |
| Голос | Блокировка | Сохранение + транскрипция |
| Аудио/документ | Блокировка | Сохранение + транскрипция |
| Фото | Блокировка | Сохранение + запрос подписи |
