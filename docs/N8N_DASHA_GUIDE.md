# Инструкция для Даши: как управлять пайплайном в n8n

> Этот документ — для Даши (продакт-менеджер). Объясняет, как менять логику пайплайна
> создания книги через n8n без помощи разработчика.

---

## Главное правило

Есть **два уровня управления** пайплайном:

| Что менять | Где менять | Требует разработчика? |
|---|---|---|
| Текст промпта агента (что пишет ИИ) | Твоя **админка** → раздел «Агенты» | ❌ не нужен |
| Порядок агентов, связи, новые роли | **n8n** (ссылка ниже) | ⚠️ проконсультируйся |

**Большинство изменений делается через промпты в админке — без n8n.**

---

## Доступ в n8n

Ссылка: **https://admin.glava.family/n8n/**

Логин и пароль — те же, что дал разработчик.

После входа ты увидишь список workflows. Нужный называется:
**«GLAVA · Phase A — Book Pipeline»**

---

## Часть 1. Менять тексты промптов (делается в Админке, не в n8n)

Это самое частое. Хочешь изменить, как Писатель пишет текст или как Корректор проверяет?

1. Зайди на **https://admin.glava.family**
2. Раздел «Агенты» (в твоём кабинете Даши)
3. Найди нужного агента (например, «Писатель»)
4. Нажми «Редактировать» → измени текст промпта → «Сохранить»

**Готово.** Следующий же запуск пайплайна возьмёт новый промпт автоматически.
Перезапускать n8n или бот **не нужно**.

---

## Часть 2. Работа с n8n — как устроен экран

Открой workflow «Phase A». Ты увидишь цепочку блоков (нод) слева направо:

```
[Webhook] → [Фактолог] → [Писатель] → [Фактчекер] → [Литредактор] → [Корректор]
                                    ↘
                               [Интервьюер] → (параллельно)
                                                          ↓
                                               [Продюсер] → [Telegram]
```

**Что такое каждый блок:**
- Синие блоки с кодом — «упаковщики» данных (трогать не нужно)
- Оранжевые HTTP-блоки — вызовы OpenAI (агенты)
- Зелёные HTTP-блоки — запросы к промптам и Telegram

---

## Часть 3. Временно отключить агента

Например, хочешь проверить пайплайн без Фактчекера.

1. Кликни правой кнопкой на нужную ноду → **«Disable»**
2. Нода станет серой — в следующем запуске она пропускается
3. Чтобы включить обратно — правая кнопка → **«Enable»**

⚠️ **Важно:** если отключаешь агента — нужно также отключить или перекинуть связи перед ним и после. Иначе цепочка прервётся. Лучше спроси разработчика.

---

## Часть 4. Поменять порядок агентов

> ⚠️ Это влияет на работу пайплайна. Рекомендуется делать с разработчиком или сделать копию workflow перед правками.

**Чтобы переключить, куда идёт результат агента:**

1. Кликни на стрелку между двумя нодами — появится крестик
2. Нажми крестик — связь удалится
3. Потяни новую связь: наведи на правый край ноды → появится точка → тяни к следующей ноде

---

## Часть 5. Добавить нового агента в пайплайн

Например, хочешь добавить «Стилевой контролёр» между Литредактором и Корректором.

### Шаг 1 — Добавь промпт в админке
В разделе «Агенты» создай новую роль, например `style_checker`. Напиши промпт.

### Шаг 2 — Добавь 3 ноды в n8n

Нажми **«+»** (Add node) в правом нижнем углу или кликни в пустом месте.

**Нода 1 — Получить промпт (HTTP Request):**
- Type: `HTTP Request`
- Method: `GET`
- URL: `http://127.0.0.1:5001/api/prompts/style_checker`
- Имя: `Get Prompt: Style Checker`

**Нода 2 — Вызов OpenAI (HTTP Request):**
- Type: `HTTP Request`
- Method: `POST`
- URL: `https://api.openai.com/v1/chat/completions`
- Headers: добавь `Authorization: Bearer {{ $env.OPENAI_API_KEY }}`
- Body — скопируй из соседнего агента (например, Корректора) и замени:
  - `Get Prompt: Style Checker` вместо `Get Prompt: Proofreader`
  - `Wrap for Style Checker` — имя ноды с входными данными (см. ниже)
  - Имя: `Style Checker`

**Нода 3 — Распаковать результат (Code):**
- Скопируй ноду `Extract from Literary Editor` — она делает то же самое
- Переименуй в `Extract from Style Checker`

### Шаг 3 — Перекинь связи

- Отсоедини стрелку от Литредактора к Корректору
- Подключи: Литредактор → Style Checker → Корректор

### Шаг 4 — Нажми Publish

Кнопка **«Publish»** в правом верхнем углу. Без этого изменения не вступят в силу.

## Часть 6. Тестирование изменений

Прежде чем применять изменения к реальным пользователям:

### Тест в режиме «Test workflow»

1. Нажми **«Test workflow»** (кнопка рядом с Publish)
2. Отправь тестовый запрос через кнопку **«Execute workflow»**
3. Каждая нода покажет зелёный (ОК) или красный (ошибка) статус
4. Кликни на ноду → увидишь Input и Output данные

### Тест через бота

Попроси разработчика отправить тестовый curl-запрос с коротким транскриптом.
Смотри результат в Telegram и в разделе **Executions** в n8n (левое меню).

---

## Часть 7. Executions — история запусков

В левом меню n8n → **«Executions»**.

Здесь видны все запуски пайплайна:
- 🟢 **Succeeded** — прошло успешно, время выполнения
- 🔴 **Error** — ошибка, кликни чтобы увидеть на какой ноде

Если у клиента что-то пошло не так — первым делом сюда.

---

## Часть 8. Чего нельзя делать без разработчика

| Действие | Почему опасно |
|---|---|
| Удалять ноду **Webhook** | Это точка входа всего пайплайна — бот перестанет работать |
| Менять путь Webhook (`glava/phase-a`) | Бот будет слать запросы на старый путь — ничего не запустится |
| Менять `OPENAI_API_KEY` в нодах | Ключ подхватывается из переменной окружения автоматически |
| Менять URL Flask API (`127.0.0.1:5001`) | Это адрес нашего сервера — менять только если сервер переехал |
| Удалять ноду **Merge: Bio + Questions** | Без неё Продюсер не получит оба результата |

## Быстрая памятка

```
Хочу изменить текст агента → Админка → Агенты → Редактировать
Хочу отключить агента временно → n8n → ПКМ на ноде → Disable
Хочу добавить нового агента → Сначала промпт в админке, потом 3 ноды в n8n
После любых изменений в n8n → нажать Publish
Что-то сломалось → n8n → Executions → найти красный запуск → кликнуть
```

пайплайн после неудачного эксперимента.
