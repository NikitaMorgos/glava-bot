# Разбивка интервью на диалог по спикерам (диаризация)

Чтобы текст интервью шёл не сплошным блоком, а с указанием **кто что сказал** (Спикер 1 / Спикер 2), нужна разметка по спикерам. Ниже — что использовать и что сделать с **Яндекс SpeechKit** и **AssemblyAI**.

---

## 1. Два рабочих варианта

| Вариант | Как получаем спикеров | Что нужно |
|--------|------------------------|-----------|
| **AssemblyAI** | Готовые метки из облака (`speaker_labels=True`) | `ASSEMBLYAI_API_KEY`. Для длинных файлов — передавать **URL** файла (presigned), а не загружать с машины. |
| **Локально: диаризация + Whisper** | Диаризация (pyannote/Resemblyzer) даёт «кто когда говорил»; Whisper даёт фразы с таймкодами → привязка по времени | `faster-whisper` или `openai-whisper`, при желании `HUGGINGFACE_TOKEN` для pyannote. |

---

## 2. Яндекс SpeechKit

- SpeechKit **не делает диаризацию** (не размечает спикеров). Он только распознаёт речь.
- В ответе **longRunningRecognize** приходят **chunks**; у каждого chunk в API v2 могут быть поля с таймкодами и **words** с `start_time` / `end_time`. По ним можно привязывать фразы к нашей локальной диаризации.

**Что бывает на практике:**

- Для длинных файлов SpeechKit часто отдаёт **один большой chunk** на всё аудио (или без нормальных таймкодов по фразам). Тогда привязывать к спикерам нечего — весь текст уйдёт к одному «спикеру».
- В коде мы так и делаем: если от SpeechKit пришло **меньше двух сегментов с таймкодами**, для разбивки по спикерам **не используем** SpeechKit, а идём в **Whisper** (он даёт много фраз с `start`/`end`).

**Что можно сделать с Яндексом:**

1. **Ничего не менять** — оставить текущую логику: при 2+ сегментах с таймкодами используем их для диаризации, иначе — Whisper.
2. **Проверить ответ API** — включить логирование сырого `response` (например, количество `chunks`, есть ли в `alternatives[0]` поле `words` с таймкодами). Если окажется, что при каком-то другом формате запроса приходит разбивка по фразам — доработать парсер в `_get_speechkit_segments_with_timing`.
3. В [документации SpeechKit v2](https://cloud.yandex.ru/docs/speechkit/stt/api/request-api) и в proto (например, `stt_service.proto`) посмотреть, есть ли опции вроде «возвращать по фразам/utterance» — если да, включить их в запрос и снова проверять формат ответа.

Итого: для **нормальной разбивки на диалог** при текущем поведении API разумно **не полагаться только на SpeechKit** для длинных интервью, а использовать **Whisper** (или AssemblyAI) как источник фраз с таймкодами/спикерами.

---

## 3. AssemblyAI

- При **`speaker_labels=True`** сервис сам возвращает реплики с метками спикеров (A, B, …). Это **готовый диалог** — ничего дополнительно выравнивать не нужно.

**Проблема:** при длинных файлах (десятки минут) загрузка файла с вашей машины в AssemblyAI может **обрываться** (таймаут/разрыв соединения).

**Что сделать:**

1. **Передавать в AssemblyAI не файл с диска, а URL** (например, presigned URL на тот же файл в S3/Object Storage). Тогда загрузку делает сам AssemblyAI с облака, а не ваш скрипт — обрывы из-за долгой загрузки с машины исчезают.
2. В коде уже есть повтор попыток и увеличенный таймаут; для длинных файлов **обязательно использовать вариант с URL** (см. ниже реализацию).

В `assemblyai_client.py` добавлена поддержка вызова с **URL**: если передан `audio_url`, используется он, иначе — загрузка файла по `audio_path`.

---

## 4. Кратко по шагам

- **Нужен диалог с спикерами без возни с диаризацией**  
  → Включить **AssemblyAI** с `speaker_labels=True` и для длинных файлов передавать **URL** (presigned), а не путь к файлу.

- **Нужна разбивка по спикерам при использовании Яндекса**  
  → Оставить текущую схему: пробуем взять из SpeechKit сегменты с таймкодами; если их мало (0–1) — используем **Whisper** (или AssemblyAI) для получения фраз с таймкодами и дальше выравниваем по локальной диаризации.

- **Whisper обязателен для локальной диаризации**  
  → Установить `pip install faster-whisper` (или `openai-whisper`), иначе при «только SpeechKit» и одном chunk разбивка по спикерам не получится.

Файлы с логикой: `transcribe.py` (диаризация, выравнивание, использование SpeechKit/Whisper), `assemblyai_client.py` (транскрипция, в т.ч. по URL).
