Пользовательские сценарии бота GLAVA (изменённая версия)

Обновлено: сценарий персонажей и оформления заказа приведён в соответствие с ТЗ

1. Основные изменения

  • Персонажи при 0: бот сразу просит ввести имя (без кнопок). После имени — родство. Только когда первый персонаж сохранён (имя+родство), появляются кнопки: «Добавить ещё», «Продолжить», «Удалить».
  • Оформление заказа: Начать → (бот просит имя) → имя → (бот просит родство) → родство → (появляются кнопки) → Продолжить → email → Итого
  • Возобновление черновика: если draft есть, но characters=[], при resume — возврат в режим ввода имени (без кнопок), а не на список с кнопками
  • Произвольный текст: текст принимается как имя/родство/email только в соответствующих state'ах FSM, иначе — «Используйте кнопки»

2. Схема потока (изменённый фрагмент)

[черновик создан]
       │
       ▼
Персонажи (0)
→ Ввод имени (без кнопок)
→ Ввод родства
       │
       ▼
Персонажи (список) + кнопки [Добавить ещё, Продолжить, Удалить]
       │
       ▼
Email для чека → Итого заказ → Перейти к оплате
