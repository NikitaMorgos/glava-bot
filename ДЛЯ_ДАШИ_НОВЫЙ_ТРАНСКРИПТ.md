# Новый транскрипт интервью для проверки

## Где взять новый вариант

**Папка с результатами:**
```
c:\Users\user\Dropbox\Public\Cursor\GLAVA\pipelines\interview_YYYYMMDD_HHMMSS\
```

Или относительно корня проекта: `GLAVA/pipelines/interview_YYYYMMDD_HHMMSS/`

(Будет создана при каждом запуске пайплайна. Ищи последнюю по дате в имени.)

**Файлы:**
- **structured.json** — полная структура: чанки, блоки по темам, реплики с разделением интервьюер/герой
- **draft_chapter.txt** — черновая глава (2–4 стр.), собранная только из фактов и эпизодов
- **comparison_report.md** — сравнение со старым вариантом

**Сплошной текст для проверки:**  
Открой `structured.json` → раздел `blocks` → в каждом блоке поле `dialogue` — там все реплики подряд.  
Или полная транскрипция: `chunks` → в каждом `clean_transcript`.

## Как сравнить

1. **Открыть старый вариант:**  
   `exports/client_605154_ddmika/transcript.txt`

2. **Открыть новый вариант:**  
   `pipelines/interview_*/structured.json`  
   - Для сплошного текста: из всех `chunks` склеить `clean_transcript`.  
   - Для структурированного вида: смотреть блоки и `dialogue` по темам.

3. **Посмотреть отчёт сравнения:**  
   `pipelines/interview_*/comparison_report.md`  
   Там: расхождения по фактам, улучшения, примеры фрагментов.

4. **Проверить черновую главу:**  
   `pipelines/interview_*/draft_chapter.txt`  
   Сборка только из фактов и эпизодов, без домысливания.

## Если папки pipelines нет или она пустая

Нужно запустить пайплайн (на компьютере с доступом к проекту и ffmpeg):

```bash
cd GLAVA
python scripts/pipeline_interview_v2.py --longest
```

Он возьмёт самое длинное интервью из БД (это 605154, ~40 мин), сделает новую транскрипцию и структуру.
